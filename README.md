# Poliklininka — система управления поликлиникой

**Poliklininka** — учебный WPF-проект на C# для работы с базой данных поликлиники.

Проект демонстрирует работу с одной предметной областью через разные технологии доступа к данным:

* **ADO.NET**;
* **Entity Framework Core**;
* **NHibernate**.

В приложении реализованы роли администратора, регистратора, врача и пациента.

---

## Технологии

* **C#**
* **WPF** — пользовательский интерфейс
* **MVVM** — архитектурный паттерн
* **PostgreSQL** — база данных
* **ADO.NET / Npgsql** — прямой доступ к базе данных через SQL-запросы
* **Entity Framework Core** — ORM для основной части приложения
* **NHibernate** — ORM для отдельной части функционала врача
* **Microsoft.Extensions.DependencyInjection** — внедрение зависимостей
* **pgAdmin** — просмотр и демонстрация схемы базы данных

---

## Назначение проекта

Приложение предназначено для автоматизации базовых процессов поликлиники:

* авторизация пользователей;
* управление пользователями;
* работа с пациентами;
* запись пациентов к врачу;
* просмотр расписания;
* работа с медицинскими картами;
* ведение истории приёмов;
* демонстрация разных способов работы с базой данных.

---

## Роли пользователей

| Роль              | Назначение                                                                                |
| ----------------- | ----------------------------------------------------------------------------------------- |
| **Администратор** | Управление пользователями, демонстрация ADO.NET                                           |
| **Регистратор**   | Запись пациентов к врачам, демонстрация Entity Framework                                  |
| **Врач**          | Работа с медкартами, историями приёмов, анализами и назначениями, демонстрация NHibernate |

# Соответствие требованиям задания

## Задание 1. Схема данных

Схема базы данных представлена в виде диаграммы связанных таблиц.

Для демонстрации используется схема из **pgAdmin**.

Основные таблицы базы данных:

* `users`;
* `patients`;
* `doctors`;
* `appointments`;
* `schedules`;
* `med_cards`;
* `visit_histories`;
* `med_services`;
* `blood_groups`;
* `allergies`;
* `chronic_diseases`;
* таблицы связей для отношений многие-ко-многим.

---

## Задание 2. ADO.NET

ADO.NET реализован в панели администратора.

Основные файлы:

* `AdminWindow.xaml`;
* `AdminViewModel.cs`;
* `IAdminAdoService.cs`;
* `AdminAdoService.cs`.

### 1. Однотабличная база данных с выпадающим списком

В панели администратора реализована работа с таблицей `users`.

Администратор может:

* просматривать пользователей;
* добавлять пользователей;
* редактировать пользователей;
* удалять пользователей.

Поле `role` редактируется через выпадающий список `ComboBox`.

Доступные роли:

* `Patient`;
* `Doctor`;
* `Admin`;
* `Registrar`.

Таким образом, требование “однотабличная база данных с выпадающим списком в одном из полей” реализовано через таблицу `users` и поле `role`.

### 2. Форма с подчинённой таблицей

В панели администратора добавлена вкладка связанных данных.

Главная таблица:

* `users`.

Подчинённая таблица:

* `appointments`.

При выборе пользователя отображаются связанные данные:

* для пациента — его записи на приём;
* для врача — его приёмы;
* для администратора и регистратора выводится сообщение об отсутствии подчинённых данных.

Работа выполнена через ADO.NET с использованием `NpgsqlConnection`, `NpgsqlCommand` и SQL-запросов.

---

## Задание 3. NHibernate

NHibernate используется в функционале врача.

Основные задачи, реализованные через NHibernate:

* работа с пациентами;
* работа с медицинскими картами;
* просмотр истории посещений;
* работа с анализами;
* работа с назначениями;
* работа с аллергиями и хроническими заболеваниями.

### 1. Однотабличная база данных с выпадающим списком

В окне врача используются выпадающие списки для выбора медицинских данных.

Примеры выпадающих списков:

* группа крови;
* аллергии;
* хронические заболевания;
* анализы;
* препараты.

### 2. Форма с подчинённой таблицей

Врач выбирает пациента или приём, после чего отображаются связанные данные:

* история посещений;
* анализы;
* назначения;
* данные медицинской карты.

Это демонстрирует связь “один-ко-многим”.

### 3. Реализация наследования

В NHibernate-части реализовано наследование пользователей.

Пример:

* базовая сущность пользователя;
* пациент;
* врач.

Пациент и врач являются специализированными типами пользователя.

### 4. Реализация композиции один-к-одному

Реализована связь пациента с медицинской картой.

Пример:

* пациент;
* медицинская карта.

У одного пациента есть одна медицинская карта.

### 5. Реализация отношения многие-ко-многим

В медицинской части реализованы отношения многие-ко-многим.

Примеры:

* медицинская карта и аллергии;
* медицинская карта и хронические заболевания.

Одна медицинская карта может содержать несколько аллергий, и одна аллергия может относиться к нескольким пациентам.

---

## Задание 3. Entity Framework

Entity Framework используется в основной модели данных и функционале регистратора.

Основные файлы:

* `ApplicationDbContext.cs`;
* EF-сущности из папки `Entities`;
* EF-сервисы;
* окно регистратора.

### 1. Однотабличная база данных с выпадающим списком

В интерфейсе регистратора используются выпадающие списки для выбора данных из таблиц базы данных.

Примеры:

* выбор пациента;
* выбор врача;
* выбор медицинской услуги;
* выбор свободного времени приёма.

### 2. Форма с подчинённой таблицей

В панели регистратора реализована работа со связанными данными.

Примеры:

* выбранный пациент → список его записей;
* выбранный врач → расписание и свободные слоты;
* выбранная услуга → создание записи на приём.

Это демонстрирует связь “один-ко-многим”.

### 3. Реализация наследования

В Entity Framework реализована иерархия пользователей.

Пример:

* `User`;
* `Patient`;
* `Doctor`.

Пациент и врач наследуются от общего пользователя.

### 4. Реализация композиции один-к-одному

В Entity Framework реализована связь пациента с медицинской картой.

Пример:

* `Patient`;
* `MedCard`.

У одного пациента может быть одна медицинская карта.

### 5. Реализация отношения многие-ко-многим

В Entity Framework реализованы связи многие-ко-многим через промежуточные таблицы.

Примеры:

* пациент / медицинская карта и аллергии;
* пациент / медицинская карта и хронические заболевания.

# Архитектура проекта

Проект построен по паттерну **MVVM**.

Общая схема работы приложения:

```text
View (XAML)
    ↕ Binding
ViewModel
    ↕ Interface
Service
    ↕
Database
```

В проекте используется сервисный слой. Окна не обращаются к базе данных напрямую, а работают через ViewModel и сервисы.

## Основные возможности

### Авторизация

* вход пользователя по логину и паролю;
* определение роли пользователя;
* открытие соответствующего окна приложения.

### Пациент

* просмотр личной информации;
* редактирование профиля;
* просмотр данных, связанных с пациентом.

### Регистратор

* выбор пациента;
* выбор врача;
* выбор услуги;
* просмотр доступного времени;
* создание записи на приём;
* просмотр записей пациента.

### Врач

* просмотр пациентов и приёмов;
* работа с медицинской картой;
* просмотр истории посещений;
* добавление медицинских данных;
* работа с анализами и назначениями.

### Администратор

* просмотр пользователей;
* добавление пользователей;
* редактирование пользователей;
* удаление пользователей;
* выбор роли через выпадающий список;
* просмотр подчинённых данных выбранного пользователя.

## База данных

В качестве СУБД используется **PostgreSQL**.

Структура базы данных включает:

* пользователей;
* пациентов;
* врачей;
* расписание врачей;
* записи на приём;
* медицинские услуги;
* медицинские карты;
* историю посещений;
* справочники медицинских данных;
* таблицы связей для отношений многие-ко-многим.

Настройки подключения находятся в файле:

```text
appsettings.json
```

---

## Запуск проекта

1. Установить и запустить PostgreSQL.
2. Открыть проект в Visual Studio.
3. Проверить строку подключения в `appsettings.json`.
4. Применить миграции или убедиться, что база данных создана.
5. Запустить приложение.
6. Войти под пользователем нужной роли.

## Итог

Проект демонстрирует работу WPF-приложения с базой данных PostgreSQL и реализует несколько подходов доступа к данным:

* ADO.NET;
* Entity Framework Core;
* NHibernate.

В рамках одной предметной области показаны:

* однотабличная форма с выпадающим списком;
* форма с подчинённой таблицей;
* наследование;
* композиция один-к-одному;
* отношение многие-ко-многим;
* работа с несколькими ролями пользователей.
